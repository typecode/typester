Module: core/Container

Container - A factory for building container classes. Containers are built up of a mediator instance that is shared with its child modules.
Parameters:
Name Type Description
containerObj object **(Required)** A descriptor of the container
Properties
Name Type Description
name string **(Required)** The name of the container
handlers object A map of mediator event strings and method names: 'event:string' : 'methodName'. Event strings are then mapped to the named method.
methods object A map of named methods.
modules array An array of modules to be instantiated by the container. [{ class: ModuleClass }]
mediatorOpts object A map of options to be passed on to the Mediator.
mediator mediator A mediator instance to act as a parent for the mediator instance created in this container.
Source:
  • core/Container.js, line 3
Returns:
- A container class that can be instantiated.
Type
function
Example
import MyModule from '../modules/MyModule.js';

const MyContainer = Container({
  name: 'MyContainer',
  modules: [
    { class: MyModule }
  ],
  handlers: {
    requests: { // Request handlers mapped to methods & registered with the mediator
      'mycontainer:request:something' : 'getSomething'
    },
    commands: { // Command handlers mapped to methods & registered with the mediator
      'mycontainer:do:something' : 'doSomething'
    },
    events: { // Event handlers mapped to methods & registered with the mediator
      'module:event' : 'handlerMethod'
    }
  },
  methods: {
    setup () {}, // A setup hook called before modules and child containers are initialized.
    init () {} // An init hook called after all modules and child containers have been initialized.

    // The rest of the methods required for this container.
    getSomething () {},
    doSomething () {},
    handlerMethod () {}
  }
});

const myContainer = new MyContainer(containerOpts);

// myContainer has a method you can use to update the parent mediator using:
myContainer.setMediatorParent(mediatorInstance);
